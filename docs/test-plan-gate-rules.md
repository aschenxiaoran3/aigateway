# 测试方案门禁专业规则说明 V2

## 1. 门禁目标

测试方案门禁用于确认模板版测试方案是否达到“结构完整、追溯闭环、可执行、可发布”的标准，同时把 AI 增强版作为专业度补充视图。

## 2. 规则层次

### 2.1 结构层

- 必须包含模板版阻断级章节
- 必须包含发布建议与门禁结论
- AI 增强版应具备增强说明、边界场景、测试数据建议、SQL / 日志 / 指标建议

### 2.2 追溯层

- 必须覆盖 PRD 要求
- 必须覆盖技术方案约束
- 必须覆盖 Coverage Graph 义务
- 不允许存在未绑定覆盖义务

### 2.3 可执行层

- 必须存在测试环境矩阵
- 必须存在测试数据策略
- 必须存在人工逐步执行步骤
- 必须存在接口级、状态级或字段级 DB 断言
- 必须存在进入准则 / 退出准则

### 2.4 质量层

- 不允许残留 `待补充`、`TODO`、`TBD`
- 不允许只列标题而无具体执行说明
- 不允许缺少发布建议

## 3. 阻断项

- 缺测试目标 / 范围 / 不在范围
- 缺风险清单
- 缺环境矩阵
- 缺测试数据策略
- 缺进入 / 退出准则
- 缺 PRD / 技术方案 / Coverage Graph 追溯
- 缺接口矩阵 / 状态迁移矩阵 / 逆向矩阵
- 缺字段级 DB 断言
- 缺发布建议
- 残留 `TODO` / `TBD` / `待补充`
- Coverage obligation 未绑定

## 4. 告警项

- 缺角色职责
- 缺资源 / 工时 / 里程碑
- 缺度量报告口径
- 缺自动化回归策略
- 缺历史缺陷复用建议
- AI 增强版缺失或内容空泛

## 5. 发布口径

- 模板版 `pass` 才允许发布
- 模板版 `warn` 不允许发布
- AI 增强版 `warn` 可接受
- AI 增强版不单独作为发布主口径
