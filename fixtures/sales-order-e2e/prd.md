# 销售订单 E2E 样板 PRD（可发布闭环样板）

## 需求背景
销售订单创建与审核流程自动化验收。

## 用户故事
- 作为业务员，我能够创建销售订单，以便记录客户意向。
- 作为审核员，我能够驳回订单，以便阻断不合规单据。
- 作为仓配协同人员，我能够在外部出库 `confirm` 回写后看到订单状态变化。

## 正向主流程
创建销售订单草稿 → 保存草稿 → 提交审核 → 审核通过 → 等待外部 WMS 子流程 `confirm` 回写 → 已确认出库 → 满足累计出库数量后闭单。

## 逆向流程
- 审核员可对 `已提交` 订单执行驳回，订单进入 `已驳回`。
- `已驳回` 订单允许编辑后重新提交。
- `已提交` 或 `已驳回` 订单允许作废，作废后进入 `已作废`。
- 已确认出库的订单不允许作废，需走补偿流程。

## 状态机与关键迁移
- 合法状态迁移：`草稿` → `已提交` → `已审核` → `已确认出库` → `已闭单`
- 合法状态迁移：`已提交` → `已驳回` → `草稿`
- 合法状态迁移：`已提交` → `已作废`
- 非法迁移：`已驳回` 不允许直接审核
- 非法迁移：`已审核` 不允许重复提交
- 非法迁移：`已确认出库` 禁止作废，必须先补偿再关闭

## 验收标准
- 主流程成功率 ≥ 99%。
- 驳回后状态不可直接审核，需重新编辑并提交。
- 出库 `confirm` 回写后，`shipped_qty` 累计等于 `order_qty` 时自动闭单。

## 角色与权限边界
- 业务员：可创建、编辑草稿、提交审核。
- 审核员：可审核、驳回、作废未出库订单。
- 仓配接口：仅可通过外部 `confirm` 子流程回写出库结果。
- 任何无审批权限角色都不允许审核或作废他人单据。

## 字段定义与字段规则
| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| order_id | string | 订单号 |
| status | string | 状态 |
| customer_id | string | 客户编号 |
| order_qty | number | 订单数量 |
| shipped_qty | number | 已出库累计数量 |
| closed_flag | boolean | 是否闭单 |

- 字段规则：`order_id` 创建后只读。
- 字段规则：`status` 只能按状态机迁移。
- 字段规则：`shipped_qty` 只能由外部子流程 `confirm` 回写。
- 字段规则：`closed_flag=true` 的前提是 `shipped_qty = order_qty`。

## 异常与提示语
- 当前订单状态不允许提交。
- 当前订单状态不允许审核。
- 已确认出库，不允许作废。
- 缺少客户编号或明细行，提交失败。

## 数据兼容性
与现有订单表扩展字段兼容，保持 `customer_id`、`status`、`shipped_qty` 和 `closed_flag` 的向后兼容。

## 范围外说明
- 一期不做自动拆单。
- 范围外：跨组织调拨和价格审批链路。
