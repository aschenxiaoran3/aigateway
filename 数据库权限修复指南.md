# AI Gateway 数据库权限修复指南

## 🐛 问题描述

AI 网关无法写入用量日志到数据库，报错：
```
Access denied for user 'erp_test'@'182.148.54.182' (using password: YES)
```

## ✅ 解决方案

### 方案 1: 在阿里云 RDS 控制台授权（推荐）

1. 登录阿里云 RDS 控制台
2. 找到实例 `rm-bp17ns75mi339poo5fo`
3. 进入"账号管理"
4. 找到账号 `erp_test`
5. 点击"修改权限"
6. 勾选以下权限：
   - SELECT
   - INSERT
   - UPDATE
   - DELETE
   - CREATE
   - ALTER
   - INDEX
7. 点击"确定"

### 方案 2: 使用 root 用户执行 SQL

如果有 root 用户权限，执行以下 SQL：

```sql
-- 给现有用户授权
GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER, INDEX 
ON aiplan_erp_test.* TO 'erp_test'@'%';
FLUSH PRIVILEGES;
```

### 方案 3: 创建新用户（已配置）

系统已配置新用户 `gateway_user`，密码请替换为你自己的强密码，例如 `CHANGE_ME_STRONG_PASSWORD`

执行 `/Users/xiaoran/.openclaw/workspace/projects/ai-platform/database/fix-permissions.sql` 创建用户并授权。
