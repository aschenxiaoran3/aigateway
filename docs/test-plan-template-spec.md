# 测试方案模板规范 V2

## 1. 目标

标准模板版测试方案是正式交付主口径，要求同时满足：

- 结构稳定
- 可审计
- 可追溯
- 可执行
- 可作为发布前门禁依据

## 2. 模板版阻断级章节

- 测试目标 / 测试范围 / 不在范围
- 测试对象 / 版本边界 / 变更范围
- 假设 / 依赖 / 约束
- 风险清单与优先级
- 进入准则 / 退出准则
- 测试环境矩阵
- 测试数据策略
- 测试生成前置契约检查
- PRD 追溯矩阵
- 技术方案追溯矩阵
- Coverage Graph / 覆盖义务矩阵
- 接口级验证矩阵
- 状态迁移矩阵
- 逆向 / 非法 / 异常场景矩阵
- 字段级 DB 断言矩阵
- 子流程 / 外部系统边界验证
- 用例汇总
- 关键用例详述
- 缺陷记录策略 / 回归策略
- 发布建议与门禁结论
- 发布前门禁结论

## 3. 模板版告警级章节

- 角色与职责
- 资源 / 工时 / 里程碑
- 度量与报告口径
- 自动化范围与回归策略
- 历史缺陷复用建议

## 4. 内容要求

- 所有覆盖义务都必须能绑定测试用例。
- 追溯矩阵必须能回溯到 PRD、技术方案和 Coverage Graph。
- 每个 P0 场景必须有可执行步骤、测试数据、预期结果和 DB / 接口 / 状态断言。
- 风险、环境、测试数据、进入/退出准则、发布建议必须明确。
- 正式发布版不允许残留 `待补充`、`TODO`、`TBD`。

## 5. AI 增强版与模板版关系

- AI 增强版只能基于模板版扩展。
- 模板版是发布主口径。
- AI 增强版建议补充：
  - 边界值与组合场景
  - 恢复路径 / 异常分支
  - 推荐测试数据集
  - SQL / 日志 / 指标断言建议
  - 高风险场景优先执行建议
