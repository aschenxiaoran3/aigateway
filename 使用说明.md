# AI 工程化管理平台使用说明

本文件作为平台入口说明，正式使用手册请优先阅读 [docs/platform-user-manual.md](./docs/platform-user-manual.md)。

## 你现在拿到的是什么

这不是单一 AI 网关，而是一套 AI 工程化管理平台，包含：

- 项目治理
- 控制平面
- 文档门禁与 `doc-pipeline-v1`
- 运行编排
- 知识与审计
- 度量与可观测
- 阶段验收

## 推荐开箱顺序

```bash
cd /Users/xiaoran/.openclaw/workspace/projects/ai-platform
npm run phase1:migrate
npm run platform:init
npm run knowledge:ingest
```

## 初始化后默认会有什么

- 标准节点、Schema、Skill、默认规则摘要
- 平台手册、测试模板规范、门禁规则说明等知识资产
- 默认演示项目与里程碑
- 默认文档任务与双轨测试方案样板
- 默认 Trace、Evidence Pack、RAG 日志、知识抽检记录
- 默认提效基线与阶段验收样例

## 文档门禁工作流

上传 PRD、技术方案、接口契约、DDL 后，可单步执行，也可一键执行：

`输入契约 -> PRD 门禁 -> 技术方案门禁 -> Coverage Graph -> 测试方案双轨生成 -> 测试方案门禁 -> 发布`

测试方案会生成三类产物：

- 标准模板版草稿
- AI 增强版草稿
- 正式发布版

发布口径：

- 标准模板版门禁必须 `pass`
- AI 增强版允许 `warn`

## 管理台推荐查看路径

1. 控制平面：看标准节点、默认规则包、关联知识资产
2. 文档门禁：看默认样板任务与双轨测试方案
3. 运行编排：按 `trace_id` 看链路推进、阻断和推荐动作
4. 知识与审计：看资产目录、入库状态、RAG 日志、抽检记录
5. 度量与可观测：看运行量、门禁通过率、知识命中率、阶段验收覆盖率
6. 阶段验收：看 Evidence Pack、提效基线和默认验收样例
